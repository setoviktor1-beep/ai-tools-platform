## EMERGENCY FIXES

### If Post Feels Generic:
1. Add specific $ amount, timeline, or metric
2. Remove first paragraph (often unnecessary)
3. Replace abstract concepts with concrete examples
4. End with honest limitation instead of advice

### If Post Feels Too Long:
1. Delete introduction (start with the insight)
2. Cut to 3 main points maximum
3. Remove repeated concepts
4. Eliminate throat-clearing phrases

### If Post Sounds Like AI:
1. Add contractions (I'm, you're, we've)
2. Break one grammar rule intentionally
3. Use industry-specific jargon naturally
4. Add personal quirk or phrase

## ARCHETYPE-SPECIFIC CHECKS

### For Contrarian Takes:
- [ ] Claim is actually controversial (not just "hot take")
- [ ] Backed by specific experience or data
- [ ] Doesn't rely on strawman argument
- [ ] Shows WHY common wisdom is wrong

### For Technical Deep Dives:
- [ ] Code/tools mentioned are real and current
- [ ] Actual technical depth (not surface explanation)
- [ ] Plain English used for complex concepts
- [ ] Includes unexpected insight, not just how-to

### For Failure Stories:
- [ ] Actual consequences stated (time/money)
- [ ] NO redemption arc or silver lining
- [ ] Lesson is broadly applicable
- [ ] Vulnerability is genuine, not performed

---

## POST-PUBLISH LEARNING

### After Posting, Track:
- Engagement pattern (who engaged most?)
- Comments revealing confusion (clarify next time)
- Sections that resonated (double down)
- Unintended interpretations (word choice)

### Red Flags in Comments:
- "Thanks for sharing!" (too generic)
- "Inspiring!" (not substantive enough)
- No debate/discussion (not thought-provoking)

### Green Flags in Comments:
- Specific questions about methodology
- Respectful disagreement with data
- Request for more technical depth
- Stories from others' similar experiences
